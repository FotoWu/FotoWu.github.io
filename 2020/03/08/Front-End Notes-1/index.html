HTML

1. Differences between http nad https

  1. HTTP+SSL => HTTPS
  2. HTTP is plaintext, so add SSL protocol to encrypt the data transferred by this protocol
  3. Use different link methods and ports(http: 80, https:443)
  4. HTTPS needs CA authentication, which is expensive
  5. HTTP is stateless
  6. HTTPS pros and cons
    • pros: security, data integrity
    • cons: expensive(SSL authentication), time-consuming, inefficient, IP resources availability(each SSL needs to bind one IP)
  7. HTTPS Working Approach

2. WebSocket

  • WebSocket is a protocol in HTML5, supporting continuous connection, while HTTP doesn’t support
  • WebSocket is based on HTTP protocol
  • WebSocket has two attributes “upgrade” and “connection”, when HTTP protocol doesn’t have
      GET /chat HTTP/1.1
  Host: server.example.com
  Upgrade: websocket
  Connection: Upgrade
  Sec-WebSocket-Key: x3JJHMbDL1EzLkh9GBhXDw==
  Sec-WebSocket-Protocol: chat, superchat
  Sec-WebSocket-Version: 13
  Origin: http://example.com

3. Web Quality

  • Accessible website for the disabled
  • Using ‘alt’ attribute to show images or sounds according to the browser type
      <img src="person.jpg"  alt="this is a person"/>

4. Cookie, sessionStorage and localStorage

  • Similarity: all stored in the browser, same root
  • Cookie
    • stored in the http request
    • small size, about 4KB
    • store user log status(set expired time)
    • track user behavior(humanity)
    • store sessionID to identify users
  • sessionStorage
    • keep until the session ends(say a browser is closed)
  • localStorage
    • keep forever
    • shared among all windows of the same root

5. Web Worker

  • independent script
  • return result by postMessage to the main thread
  • won’t block the main thread

6.Cookie Prevent XSS Attack

  • Add set-cookie to the http header
  • httponly: ban access to cookie from javascript
  • secure: only send cookie when the request is https

7. RESTful

  • locate resources using URL
  • describe operations using HTTP

8. addEventListender()

  • event => the name of event
  • function => the function to execute after the event is triggerred
  • useCapture => the event execute in capture or bubble phase
      addEventListener(event, function, useCapture)

CSS

1. Box Model

  • box-sizing：border-box,padding-box,content-box.

  • Width definition decides the type of box model

  • Standard Box Model(W3C): content box

  • IE Box Model: border box

2. Transition and Animation

  • Both change elements by time
  • Transition changes elements only when an event is triggerred, but animation doesn’t need a triggerred event
  • Transition: 2 frames(?) from…to
  • Animation: 1 frame

3. Flex Layout

  • container attributes and item attributes
    • container attr
      • flex-direction
      • flex-wrap
      • flex-flow
      • justify-content: horizontal
      • aligh-items: vertical
    • item attr
      • order
      • flex-grow
      • flex-shrink
      • flex-basis
      • flex: abbr for {flex-grow, flex-shrink, flex-basis}, default by {0, 1, auto}

JavaScript

1. Definition

  • a dynamic, weakly typed, object-orientation interpreted language
  • contains three parts
    • ECMAScript: provide kernel function
    • DOM: Document Object Model, provide methods and interfaces to access and operate webpages
    • BOM: Browser Object Model, provide methods and interfaces to interact with the browser

2. Features

  • Case sensitive
  • Reserved Words
    • KeyWord
    • FutureReservedWord
    • NullLiteral
    • BooleanLiteral
