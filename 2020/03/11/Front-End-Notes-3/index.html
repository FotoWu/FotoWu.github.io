JavaScript

0. Datatypes

1. clientHeight, scrollHeight and offsetHeight

a. clientHeight: visible block not including border and scroll-bar
b. offsetHeight: visible block including border and scroll-bar
c. scrollHeight: all blocks including the hidden part because of the scroll-bar
d. clientTop: the height of border, default by 0
e. scrollTop: the height of the hidden part because of the scroll-bar

2. How to load JS asynchronously?

a. add “defer” to <script> tag to delay the request of explanation of script

b. use async and HTML5 externally

c. the priority: defer > async

4. eval() and with

a. eval(): parse strings into javascript and execute
- inefficient
- scope: the context using it
- can’t analyze dynamic code
- ‘use strict’: can’t define a variable or function using eval()

b. with:simplify multiple operations of the same object
- can modify the existing attributes of object
- can’t create new attributes
- inefficient
- hard to debug
- likely to cause variable leak
- ‘use strict’: banned

5. CommonJS and AMD

a. CommonJS
- module.exports()
- require
- define module synchronously

b. AMD(Asynchronous Module Define): solves two main issues:
- dependencies of multiple files
- stop rendering when loading files, so no response
- e.g. requireJS

//define module
define(['dependency'], function(){
var name = 'Byron';
function printName(){
console.log(name);
}
return {
printName: printName
};
});

//load module
require(['myModule'], function (my){
my.printName();
}

8. Deep Clone

a. object or array

function deepClone(obj){
    var newObj = obj instanceOf Array ? [] : {};
    for(var i in obj){
        newObj[i] = typeOf  obj[i] == 'object' ? deepClone(obj[i]) : obj[i];
    }
    return newObj;
}

b. Primitive types of encapsulated class: valueOf()

function baseClone(base){
return base.valueOf();
}
//Number
var num=new Number(1);
var newNum=baseClone(num);
//newNum->1
//String
var str=new String('hello');
var newStr=baseClone(str);
// newStr->"hello"
//Boolean
var bol=new Boolean(true);
var newBol=baseClone(bol);
//newBol-> true

c. Date: define clone using prototype

Date.prototype.clone=function(){
return new Date(this.valueOf());
}
var date=new Date('2010');
var newDate=date.clone();

d. RegExp

RegExp.prototype.clone = function() {
var pattern = this.valueOf();
var flags = '';
flags += pattern.global ? 'g' : '';
flags += pattern.ignoreCase ? 'i' : '';
flags += pattern.multiline ? 'm' : '';
return new RegExp(pattern.source, flags);
};
var reg=new RegExp('/111/');
var newReg=reg.clone();
//newReg->  /\/111\//

2. Block and inline element

a. block: take up at least one line alone, can set up width, height, margin and padding

b. inline: doesn’t take up one line alone, can’t set up width and height, vertical margin and vertical padding

3. visibility=hidden, opacity=0, display:none

a. visibility=hidden: hide the element, not change the layout

b. opacity=0: hide the element, not change the layout

c. display:none: hide the element, change the layout, like deleting the element

4. Position

a. fixed: fixed position to browser, have no business with DOM, can be overlapped with other elements

b. relative: set up a starting point, then move according to the relative position to its starting point, can be overlapped with other elements

c. absolute: relative to its parent element, have no business with DOM, can be overlapped with other elements

d. sticky: combination of relative position and fixed position => before element arrives the threshold, relative position => after, fixed position

e. static: no position, element in DOM

6. What is BFC?

ANS: Block Formatting Context
-Definition: A block formatting context contains everything inside of the element creating it that is not also inside a descendant element that creates a new block formatting context.
-One element can’t exist in two different BFC at the same time.
-In the following code, #div_1 creates a .BFC_1, containing #div_2, #div_3, #div_4, #div_5. Because #div_5creates another BFC_2, #div_6 and #div_7 are excluded from .BFC

<div id='div_1' class='BFC_1'>
    <div id='div_2'>
        <div id='div_3'></div>
        <div id='div_4'></div>
    </div>
    <div id='div_5' class='BFC_2'>
        <div id='div_6'></div>
        <div id='div_7'></div>
    </div>
</div>

7. z-index

a. specifies the stack order of an element.

b. the greater stack order, the more the element is in front of others

c. only works on positioned elements (position: absolute, position: relative, position: fixed, or position: sticky).
